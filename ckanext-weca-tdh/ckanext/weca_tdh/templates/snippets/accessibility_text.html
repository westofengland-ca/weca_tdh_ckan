The Transport Data Hub Data Catalogue is an instance of CKAN, an open source data portal platform, and has been developed following WCAG 2.2 (AA) accessibility guidelines.

The Web Content Accessibility Guidelines (WCAG) are a set of internationally recognised guidelines to ensure that web content is accessible. WCAG provides a comprehensive and widely accepted framework for creating accessible websites and digital content.


Accessibility features

GOV.UK Design System (GDS)

The Data Catalogue was developed using the GOV.UK Design System, a comprehensive framework designed to create user-friendly, accessible, and consistent digital services. By aligning with this widely recognised and government-endorsed design system, we ensure that our website follows best practices in usability and accessibility.

To learn more, visit the GOV.UK Design System website.

Alternative text

We provide alternative text for images throughout our website. Alternative text, or "alt text," is descriptive text associated with images, ensuring that users with visual impairments or those using text-only browsers can understand the content and context of each image.

Colour contrast

The colour scheme was developed with WCAG 2.2 (AA) requirements in mind. This helps improve the experience for people with vision disabilities.

Keyboard navigation

We are committed to supporting keyboard navigation to ensure that all users, including those with motor disabilities, can navigate our website without relying on a mouse. Our site is designed to be fully accessible via keyboard input, allowing users to tab through interactive elements and access all content seamlessly.

Links & landmarks

The website incorporates accessible navigation features, including clear and descriptive link text. Links are designed to be easily distinguishable, providing meaningful information about the linked content. Additionally, we utilise landmarks to enhance navigation, allowing users to efficiently move through and understand the structure of our web pages.

Screen readers

To enhance the experience for users with visual impairments, our website is compatible with screen readers. Screen readers convert digital text into synthesised speech, providing auditory information about the content and structure of each page. We have structured our website to be screen reader-friendly, enabling an inclusive browsing experience for all users.

Compatibility

Browsers

The Data Catalogue was developed to support multiple browsers, including Google Chrome, Firefox, and Microsoft Edge.

Devices

The Data Catalogue supports devices with varying screen sizes, including mobile devices. This ensured that elements of the web page are scaled correctly and offer a consistent functional experience.

Testing

A combination of manual and automated accessibility testing tools were used, including:

Manual tests

  • Browser inspection tools (DevTools)
  • Keyboard navigation
  • Non-visual navigation with a screen reader (NVDA)

Automated tests

  • Lighthouse
  • WAVE
  • WCAG Accessible Web helper

Review and update policy

We're committed to regularly reviewing and updating the accessibility statement as the Data Catalogue evolves.

Known issues

Unfortunately, the Data Explorer preview feature for dataset resources does not currently conform to WCAG 2.2 guidelines.

Third-party content

The TDH uses Cookie Control and Google Analytics third-party services. We may not have control over the accessibility of those elements.
